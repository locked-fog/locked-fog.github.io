## 安装

使用 `WSL 2` 安装，出于方便起见，我使用了以下命令进行安装：

`wsl --install -d archlinux --name Arch --location F:/ArchonWSL/Arch` 

这将会在对应路径下创建一个虚拟磁盘并安装好 `ArchWSL` 镜像。接下来使用 `wsl -d Arch` 进入 `Arch Linux`  .

## 初始化

 `WSL` 默认使用 `root` 身份进入 `Arch` 。我首先创建了用于本地登录的用户组 `local` 和用于 `ssh` 登录的用户组 `remote` ，并分别新建了用户 `Locked_Fog` 和 `Remote_Fog` 。

为了编辑用户登录文件，首先需要安装一个文本编辑器。出于习惯我选择 `vim/gvim` ，首先需要通过 `pacman -Syu` 初始化并更新包管理器，通过 `pacman -Ss gvim` 找到包名称，然后通过 `pacman -Syu extra/gvim` 安装。

但是上述工作因为网络问题失败，由于当前版本的 Arch 没有预装任何文本编辑器，我决定将配置文件复制到 `Windows` 环境下，手动编辑后覆盖回源文件。

切换到 `/mnt/f/ArchonWSL/` ，并输入 `cp /etc/wsl.conf .` ，在当前目录下就能看到 `wsl.conf` ，在 `Windows` 中打开并附加以下行：

```conf
[user]
default=Locked_Fog
```

执行 `cp wsl.conf /etc/wsl.conf` 覆盖原文件，就能将默认用户更改为 `Locked_Fog` ，接下来使用 `exit` 退出 `Arch` ，使用 `wsl --shutdown` 关闭虚拟机，然后重新进入，即为 `Locked_Fog` 用户身份。

同时，为了解决网络问题，切换到校园网联合镜像站。修改 `/etc/pacman.d/mirrolist` ，在其最顶端加入：

```conf
Server = https://mirrors.cernet.edu.cn/archlinux/$repo/os/$arch
```

然后使用 `pacman -Syyuu` 刷新缓存。接下来应当可以解决绝大多数情况的网络问题了。

接下来安装 `sudo` ，使用命令 `pacman -Syu core/sudo` 安装。然后使用 `visudo` 编辑 `sudoers` 配置文件，取消注释 `%wheel      ALL=(ALL:ALL) ALL` ，并将自己的本地账户加入 `wheel` 用户组。在此之后一切非必要操作应当切换到普通用户操作，当需要临时管理员权限时使用 `sudo` 。

出于安全考虑，只允许本地用户使用 `sudo` 。
